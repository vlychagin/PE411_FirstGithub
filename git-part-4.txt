   
Созданный локальный репозиторий можно скопировать на удаленное хранилище репозиториев

Связь с удаленным репозиторием (удаленным хранилищем репозиториев)

git remote add origin https://github.com/vlad-lychagin-65/FirstGithub.git

Первая запись/проталкивание на удаленный репозиторий
git push -u origin master
         -u запомнить, что передача ведется с ветки master на удаленный репозиторий
===========================================
Настроить github на именование главной ветви по Вашему локальному репозиторию
Создать репозиторий без параметров
===========================================
!! создание локального репозитория и отправка его на удаленное хранилище
echo "# FirstGithub" >> README.md
git init
git add README.md
git commit -m "first commit"
git branch -M master
git remote add origin https://github.com/vlychagin/FirstGithub.git
git push -u origin master
===========================================

===========================================
Отправка существующего репозитория на удаленное хранилище
git remote add origin https://github.com/vlychagin/FirstGithub.git
git branch -M master
git push -u origin master
===========================================
последующие записи/проталкивания на удаленный репозиторий    
git push

Типичный алгоритм работы по отправке изменений
git add .                       или одной командой git commit -a -m'имяКоммита'
git commit -m'имя коммита'
git push

Клонирование удаленного репозитория на другой компьютер
git clone ссылкаНаРепозиторий
git clone https://github.com/vlychagin/FirstGithub.git

перейти в папку проекта с репозиторием

запросить изменения с удаленного репозитория (pull-request)
git pull
